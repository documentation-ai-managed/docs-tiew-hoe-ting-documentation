## Upgrading

Follow these steps to upgrade Tiew Hoe Ting Documentation smoothly.

<Steps>
  <Step title="Check Compatibility" icon="check-circle">

    Review changelog for breaking changes.

  </Step>
  <Step title="Update Package" icon="download">

    Use your preferred package manager.

    <Tabs>
      <Tab title="npm" icon="package">
        ```bash
        npm update @tiewhoeting/docs@latest
        ```
      </Tab>
      <Tab title="yarn" icon="package">
        ```bash
        yarn upgrade @tiewhoeting/docs@latest
        ```
      </Tab>
      <Tab title="pnpm" icon="package">
        ```bash
        pnpm update @tiewhoeting/docs@latest
        ```
      </Tab>
    </Tabs>

  </Step>
  <Step title="Rebuild Docs" icon="rocket">

    Clear cache and rebuild.

    <CodeGroup tabs="CLI,Docker">
      ```bash
      npm run build
      ```
      ```dockerfile
      docker build -t tiewhoeting/docs .
      docker run -p 3000:3000 tiewhoeting/docs
      ```
    </CodeGroup>

  </Step>
  <Step title="Verify Changes" icon="shield">

    Test key components like `<Tabs>` and `{tags}` escaping.

  </Step>
</Steps>

<Expandable title="Advanced Migration" default-open="false">

For projects using custom themes, update your config:

```javascript
// config.js
export default {
  theme: {
    brandColor: '#3B82F6',
    components: {
      Update: { showTags: true }
    }
  }
};
```

</Expandable>

<Callout kind="tip">
  After upgrading, validate your MDX files for parsing errors, especially nested code blocks.
</Callout>
